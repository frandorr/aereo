<objective>
Parse WMO OSCAR instrument channel characteristics from each instrument's web page and store as one JSON file per instrument.

Output: `scripts/scrape_wmo_oscar_channels.py` and `components/aer/data/wmo_oscar_instruments/*.json`.
</objective>

<context>
## URL pattern
- Base: `https://space.oscar.wmo.int/instruments/view/{acronym_lowercase}`
- Acronym may contain spaces → URL-encode with `urllib.parse.quote`

## HTML structure
- Channel table is inside `<div class="frequencytable">`
- Table columns vary by instrument type (radiative vs SAR vs lidar etc)
- First row is always the header, remaining rows are data
- Some instruments have NO frequencytable div — skip those

## Output format (per instrument JSON)
```json
{
  "instrument_id": 3,
  "instrument_acronym": "ABI",
  "instrument_fullname": "Advanced Baseline Imager",
  "url": "https://space.oscar.wmo.int/instruments/view/abi",
  "columns": ["Central wavelength", "Bandwidth", "SNR or NEΔT @ specified input", "Resolution (s.s.p.)"],
  "channels": [
    {"Central wavelength": "470 nm", "Bandwidth": "40 nm", "SNR or NEΔT @ specified input": "300 @ 100 % albedo", "Resolution (s.s.p.)": "1.0 km"},
    ...
  ]
}
```
</context>

Create `scripts/scrape_wmo_oscar_channels.py`:

1. Read `components/aer/data/wmo_oscar_instruments.csv` with `csv.DictReader`
2. For each instrument:
   - Build URL: `https://space.oscar.wmo.int/instruments/view/{quote(acronym.lower())}`
   - Fetch with `httpx` (timeout=30)
   - Parse with `beautifulsoup4`
   - Find `<div class="frequencytable">` — if absent, skip
   - Extract table header row as column names
   - Extract data rows as channel dicts
   - Save JSON to `components/aer/data/wmo_oscar_instruments/{acronym_slug}.json`
3. Rate limit: `time.sleep(0.5)` between requests
4. Error handling: try/except per instrument, log and continue
5. Progress: print every 50 instruments
6. Support `--limit N` flag for testing

Run with: `uv run python scripts/scrape_wmo_oscar_channels.py`
