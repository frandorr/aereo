# Plan 05-02: Migrate aer-search-earthaccess to pluggy hookimpl

## Objective

Rewrite the `search_earthaccess` plugin from the deprecated `@plugin(name=..., category="search")` + `SearchQuery` pattern to the new `@hookimpl` class pattern defined in `aer.plugin.core.AerSpec`.

### Key Changes

**Old signature:**
```python
@plugin(name="earthaccess", category="search")
def search_earthaccess(query: SearchQuery) -> GeoDataFrame["SearchResultSchema"]:
```

**New signature:**
```python
class EarthAccessSearchPlugin:
    @hookimpl
    def search(self, collections: list[str], intersects: GeomLike | None,
               time_range: TimeRange | None, search_params: dict | None = None) -> GeoDataFrame[SearchResultSchema]:
```

**Schema change:** The old `SearchResultSchema` had grid-exploded columns (row, col, epsg, channel, product_id, granule_id, etc.). The new one has: `id`, `collection`, `geometry`, `start_time`, `end_time`, `href`.

**Removed concepts:**
- `Product` type → replaced by `collections: list[str]` (e.g., `["VNP02IMG", "MOD02QKM"]`)
- `Channel` type → not part of search anymore
- `SearchQuery` → replaced by individual args to hookspec
- `GridSpatialExtent` → spatial filtering done via `intersects: GeomLike | None`
- `@plugin` decorator → replaced by `@hookimpl`
- `SearchResultSchema.from_grid_cell()` class method → no longer exists

<action>
Replace the entire `core.py` with a class-based plugin.

1. **Remove deprecated imports:**
   - Remove: `from aer.plugin import plugin`
   - Remove: `from aer.search import SearchQuery, SearchResultSchema`
   - Remove: `from aer.spatial import GridSpatialExtent`

2. **Add new imports:**
   ```python
   from aer.plugin import hookimpl, SearchResultSchema
   ```

3. **Keep these pure utility functions (they don't depend on deprecated types):**
   - `NoSpatialMetadataError` class
   - `_parse_umm_polygon()` function — still needed for geometry extraction

4. **Remove deprecated helper functions:**
   - Remove `_calculate_grid_cells()` — grid logic is extraction's responsibility
   - Drastically simplify `_granule_to_rows()` — no more cell × channel explosion

5. **Create class `EarthAccessSearchPlugin`** with a single method:
   ```python
   class EarthAccessSearchPlugin:
       @hookimpl
       def search(
           self,
           collections: list[str],
           intersects: GeomLike | None,
           time_range: TimeRange | None,
           search_params: dict | None = None,
       ) -> GeoDataFrame[SearchResultSchema]:
   ```

6. **Map parameters inside the method body:**
   - `collections` → passed to `earthaccess.search_data(short_name=collections, ...)`
   - `time_range` → converted to `(start_str, end_str)` tuple for `temporal` param
   - `intersects` → if it's a geometry, compute its bounds for `bounding_box` param. Use `intersects.bounds` to get `(minx, miny, maxx, maxy)`. It's a `GeomLike` (Shapely BaseGeometry or GeoJSON dict).
   - `search_params` → merge into earthaccess kwargs (e.g., `count`, `cloud_cover`, etc.)

7. **Build result rows with new schema columns — one row per granule:**
   - `id` = `meta["native-id"]` or `meta["concept-id"]`
   - `collection` = `coll_ref["ShortName"]` from UMM
   - `geometry` = polygon from `_parse_umm_polygon()`, or `intersects` as fallback
   - `start_time` = from UMM `TemporalExtent.RangeDateTime.BeginningDateTime`
   - `end_time` = from UMM `TemporalExtent.RangeDateTime.EndingDateTime`
   - `href` = first S3 direct link, or first HTTPS link (prefer S3)

   No more grid cell columns, no more channel explosion. Each granule = one row.

8. **Optionally preserve extra columns** (schema is `strict=False`):
   - `https_url` = HTTPS download link
   - `size_mb` = granule size

9. **Handle empty results:** Return empty GeoDataFrame with required columns:
   ```python
   gdf = gpd.GeoDataFrame(columns=["id", "collection", "geometry", "start_time", "end_time", "href"], geometry="geometry")
   ```

10. **Remove the old `_prepare_search_params()` and `_granule_to_rows()` functions** — their logic is inlined/simplified into the class method.
    Or keep `_parse_umm_polygon()` as a module-level utility and a small `_granule_to_row()` (singular, not plural) returning a single dict per granule.
</action>
